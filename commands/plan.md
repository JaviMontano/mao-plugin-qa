# /pqa:plan

Generate a complete architecture plan and MOAT asset strategy for a plugin. This is the entry point for the PLAN movement.

## Usage

```
/pqa:plan [plugin-brief-path]
```

- `plugin-brief-path` -- Path to a plugin concept brief generated by `/pqa:ideate`. If omitted, searches the current directory for a `plugin-brief.md` file.

## Execution Flow

1. Run `plan-architecture` -- generates agent roles, skill-movement mapping, hook strategy, command routing, quality gates, Mermaid diagrams.
2. Run `plan-moat-strategy` -- assesses complexity per skill, allocates MOAT depth, plans asset content.

Both plans are presented for review before any files are written.

## Output

- Architecture plan document (`architecture-plan.md`)
- MOAT strategy document (`moat-strategy.md`)

## Next Steps

After plan approval, run `/pqa:design` to create detailed component specifications.
